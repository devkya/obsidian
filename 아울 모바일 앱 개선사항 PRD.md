## 1. 목적 (Objective)

### 비즈니스 목적

아울 서비스 안정화를 통한 납품 일정 준수 및 서비스 품질 향상

### 사용자 가치

- **안정적인 자막 서비스 이용 경험 제공**: 공연 관람 중 끊김 없는 자막 서비스로 완전한 공연 몰입감 제공
- **공연 관람 중 끊김 없는 자막 재생 보장**: 실시간 동기화된 자막으로 놓치는 대사 없이 공연 감상 가능
- **직관적이고 편리한 앱 사용성 확보**: 복잡한 설정 없이 간단한 터치로 자막 서비스 이용 가능

## 2. 서비스 시나리오 (Service Scenarios)

### 주요 사용 시나리오

1. **기기 렌탈 및 사전 설정**: 공연(제작)사가 기기 렌탈 시 해당 기기에 대해 연결 및 티켓 사전 발급
2. **공연장 입장 및 기기 수령**: 관객이 공연장에서 사전 설정된 모바일 앱과 스마트안경 수령
3. **공연 및 언어 선택**: 앱에서 해당 공연 선택 후 자막 언어 선택 
4. **자막 서비스 이용**: 공연 중 스마트안경을 통한 자막 확인
5. **공연 종료 및 기기 반납**: 서비스 종료 후 기기 반납

## 3. 사용자 스토리 (User Stories)

### 핵심 사용자 여정

**메인 플로우**: 앱 실행 → 공연 선택 → 스마트안경 연결 → 언어 선택 → 자막 재생 → 서비스 종료

## 5. 정책 및 제약사항 (Policies & Constraints)

### 시간/수치 정책

|정책 항목|기준|비고|
|---|---|---|
|공연/언어 선택 응답시간|3초 이내|터치 후 화면 전환까지|
|자막 동기화 지연|1초 이내|실제 대사와 자막 표시 간격|
|배터리 최적화|4시간 이상 사용 가능|연속 사용 기준|
|앱 실행 시간|5초 이내|앱 아이콘 터치 후 메인 화면까지|

### 권한/접근 정책

- **키오스크 모드 해제 권한**: 관리자 권한 필요 → 일반 사용자 해제 방지
- **앱 삭제 방지 정책**: 시스템 레벨 보호 적용
- **화면 밝기 제어 권한**: 자막 재생 중 자동 밝기 조절 권한 부여

### 데이터/개인정보 정책

- **사용 로그 수집**: 서비스 개선 목적으로 최소한의 사용 데이터 수집
- **기기 정보 수집**: 오류 진단을 위한 기기 상태 정보 수집

## 6. 비기능 요구사항 (Non-functional Requirements)

### 성능 요구사항

- **앱 실행 시간**: 5초 이내
- **공연/언어 선택 성공률**: 99% 이상
- **자막 재생 안정성**: 끊김 없는 연속 재생 (99.9% 업타임)
- **메모리 사용량**: 500MB 이하 유지

### 보안 요구사항

- **키오스크 모드 보안 강화**: 무단 해제 방지 메커니즘
- **앱 무단 삭제 방지**: 시스템 권한 보호
- **데이터 암호화**: 사용자 선택 정보 암호화 저장

### 사용성 요구사항

- **직관적인 UI/UX**: 별도 설명 없이 사용 가능한 인터페이스
- **다국어 지원**: 한국어, 중국어, 일본어, 영어 지원
- **접근성 고려**: 시각/청각 장애인 접근성 준수
- **반응형 디자인**: 다양한 화면 크기 대응

### 확장성 요구사항

- **동시 접속 지원**: 공연장당 최대 500명 동시 접속
- **다중 공연 지원**: 동시 진행 공연 지원 가능한 아키텍처
- **실시간 모니터링**: 서비스 상태 실시간 모니터링 시스템

## 7. 개발 로드맵 및 우선순위 (Development Roadmap & Priorities)

### Phase 1 (P0 - Critical)

**목표**: 안정화 필수 사항 완료  
**기간**: 즉시 ~ 수요일 시연 전  
**주요 기능**:

- 공연/언어 선택 로직 수정
- 강제 밝기 제어 개선
- 팝업 로직 수정
- 화면 상태 동기화
- 키오스크 모드 안정화
- 배터리 효율화

**마일스톤**: 수요일 정부 시연 성공

### Phase 2 (P1 - High)

**목표**: 서비스 품질 향상  
**기간**: 수요일 시연 후 ~ 금요일 납품 전  
**주요 기능**:

- 터치 지연 해결
- 설명서 팝업 및 아이콘 추가
- 자막 위치/크기 설정 기능

**마일스톤**: 금요일 뷰티풀라이프 20대 납품 완료

### Phase 3 (P2 - Medium)

**목표**: 사용자 편의성 개선  
**기간**: 추후 업데이트  
**주요 기능**:

- 새로고침 영역 확대
- 시각적 가이드 개선
- 개별 밝기 설정 기능
- 터치 영역 확대

**마일스톤**: 차기 버전 업데이트

## 8. 의사결정해야 할 사항 (Decision Items)

### 구현 방식 결정

- [ ] **배터리 효율화 구체적 구현 방법**
    
    - CPU 클록 조절 vs 백그라운드 프로세스 최적화 vs 화면 밝기 자동 조절
    - → 개발팀과 기술적 검토 후 결정 필요
- [ ] **자막 위치/크기 설정 UI 디자인**
    
    - 슬라이더 방식 vs 드래그 방식 vs 프리셋 선택 방식
    - → UX 팀과 사용성 테스트 후 결정
- [ ] **설명서 팝업 표시 조건 및 빈도**
    
    - 최초 실행 시에만 vs 매번 표시 vs 사용자 선택
    - → 사용자 피드백 기반 결정

### 비즈니스 정책 결정

- [ ] **키오스크 모드 해제 권한 범위**
    
    - 관리자만 vs 공연장 직원 vs 특정 조건 충족 시
    - → 보안 정책 및 운영 효율성 고려하여 결정
- [ ] **자막 밝기 기본값 설정**
    
    - 공연장 조명 환경별 최적값 설정 기준
    - → 공연장별 테스트를 통한 데이터 수집 후 결정
- [ ] **연결 해제 시 사용자 가이드 방식**
    
    - 팝업 vs 화면 전환 vs 음성 안내
    - → 사용자 경험 최적화 관점에서 결정

### 우선순위 조정

- [ ] **P1 기능 중 납품 전 필수 포함 여부**
    
    - 설명서 기능의 금요일 납품 포함 여부
    - → 개발 진행 상황에 따라 조정
- [ ] **P2 기능의 다음 업데이트 일정**
    
    - 차기 버전 업데이트 일정 및 범위
    - → 사업 일정 및 리소스 고려하여 결정
- [ ] **긴급 수정사항 발생 시 대응 방안**
    
    - 핫픽스 배포 프로세스 및 승인 라인
    - → 운영 정책 수립 필요
