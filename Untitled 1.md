# MICE 번역 서비스 사용자 스토리

## 1. 회원가입 및 로그인 관련 스토리

### 신규 사용자 회원가입

- 모든 사용자는 이메일 회원가입을 할 수 있어야 한다
- 모든 사용자는 소셜 로그인으로 회원가입을 할 수 있어야 한다
- 모든 사용자는 일일 게스트로 간편가입을 할 수 있어야 한다
- 신규 사용자는 이름을 입력해야 한다
- 신규 사용자는 소속(회사명/기관명)을 입력해야 한다
- 이메일 회원가입 사용자는 이메일 인증을 완료해야 한다
- 모든 사용자는 전시회 키워드 기반 관심사를 선택할 수 있어야 한다
- 모든 사용자는 본인 목소리 필터 기능을 위해 음성을 등록할 수 있어야 한다
- 사용자는 관심사 선택을 나중에 설정할 수 있어야 한다
- 사용자는 음성 등록을 나중에 설정할 수 있어야 한다

### 소셜 로그인 사용자

- 소셜 로그인 사용자는 추가 정보(소속, 관심사, 음성등록)를 입력해야 한다
- 소셜 로그인 사용자는 지원되는 플랫폼을 선택할 수 있어야 한다

### 기존 사용자 로그인

- 기존 사용자는 이메일/비밀번호로 로그인할 수 있어야 한다
- 기존 사용자는 소셜 로그인으로 로그인할 수 있어야 한다

### 일일 게스트

- 일일 게스트는 이름과 소속만 입력하여 간편가입할 수 있어야 한다

## 2. 공통 기능 관련 스토리

### 전시회 선택

- 모든 사용자는 로그인 후 전시회를 선택해야 한다
- 사용자는 진행중인 전시회 목록을 확인할 수 있어야 한다
- 사용자는 전시회를 검색할 수 있어야 한다

### 앱 현지화

- 모든 사용자는 앱 인터페이스를 다국어로 사용할 수 있어야 한다
- 사용자는 앱 언어를 변경할 수 있어야 한다

### PC 앱 다운로드

- 호스트는 홈페이지에서 PC앱(윈도우/맥북)을 다운로드할 수 있어야 한다

### 자동 로그아웃

- 시스템은 일정 시간 후 자동으로 로그아웃해야 한다
- 사용자는 자동 로그아웃 시간을 설정할 수 있어야 한다
- 사용자는 자동 로그아웃 예정 알림을 받을 수 있어야 한다

### 내 정보 관리

- 사용자는 내 정보를 수정할 수 있어야 한다
- 사용자는 프로필 이미지를 업로드할 수 있어야 한다
- 사용자는 소개글을 작성할 수 있어야 한다
- 사용자는 음성을 재등록할 수 있어야 한다
- 사용자는 관심사를 수정할 수 있어야 한다

### 본인 목소리 필터

- 사용자는 회원가입 시 목소리를 등록해야 한다
- 호스트는 방 생성 시 본인 목소리 필터 기능을 ON/OFF할 수 있어야 한다
- 시스템은 다중 화자 환경에서 본인 목소리만 필터링할 수 있어야 한다

## 3. 1:N 서비스 (포럼/컨퍼런스) 관련 스토리

### 방 생성 및 관리

- 호스트는 포럼방을 생성할 수 있어야 한다
- 호스트는 사전 생성된 포럼방에 접속할 수 있어야 한다
- 호스트는 방 참여용 QR 코드를 생성할 수 있어야 한다

### 다중 발화자 식별

- 시스템은 오디오 믹서를 통해 채널별로 구분된 음성 입력을 받을 수 있어야 한다
- 시스템은 각 발화자를 익명으로 구분 표시할 수 있어야 한다
- 호스트는 다중 발화자 상태를 확인할 수 있어야 한다

### 듀얼모드

- 호스트는 듀얼화면 시 자막표출화면을 투영할 수 있어야 한다
- 호스트는 듀얼화면 자막의 글자크기를 조정할 수 있어야 한다
- 호스트는 듀얼화면 자막의 색상을 조정할 수 있어야 한다
- 호스트는 PPT 전체화면에 번역 자막을 제공할 수 있어야 한다

### 마이크 관리

- 호스트는 마이크 상태를 확인할 수 있어야 한다
- 시스템은 마이크 변경 시 알림을 표시해야 한다
- 시스템은 마이크 상태를 아이콘 컬러나 상태 변경으로 표시해야 한다

### 채팅 및 질문

- 게스트는 채팅으로 질문할 수 있어야 한다
- 게스트는 음성으로 질문할 수 있어야 한다
- 호스트는 게스트의 질문을 확인할 수 있어야 한다

### 미팅방 정보

- 사용자는 미팅방 시작 시간을 확인할 수 있어야 한다
- 사용자는 미팅방 종료 시간을 확인할 수 있어야 한다

### 자동 언어 감지

- 시스템은 같은 미팅방에서 다른 언어로 말했을 시 자동으로 언어를 변경해야 한다
- 사용자는 언어 자동 감지 기능을 ON/OFF할 수 있어야 한다

### 세션 관리

- 호스트는 세션을 생성할 수 있어야 한다
- 호스트는 세션을 수정할 수 있어야 한다
- 호스트는 세션을 삭제할 수 있어야 한다
- 호스트는 세션 순서를 변경할 수 있어야 한다
- 호스트는 완료된 세션을 재생성할 수 있어야 한다
- 시스템은 완료된 세션으로 돌아갈 수 없도록 제한해야 한다

### 동시접속자 관리

- 호스트는 동시접속자 수를 확인할 수 있어야 한다
- 시스템은 동시접속자 수가 임계값을 초과하면 컬러 및 볼드체로 표시해야 한다

### 게스트 참여

- 게스트는 QR 코드를 스캔하여 방에 참여할 수 있어야 한다
- 게스트는 방 코드를 직접 입력하여 참여할 수 있어야 한다
- 게스트는 실시간 번역을 수신할 수 있어야 한다
- 게스트는 모바일앱으로 참여할 수 있어야 한다
- 게스트는 스마트안경으로 참여할 수 있어야 한다

## 4. 1:1 서비스 (비즈니스 미팅) 관련 스토리

### 전사기록 관리

- 시스템은 호스트 및 게스트 간의 대화기록을 저장해야 한다
- 시스템은 상대방 정보를 저장해야 한다
- 시스템은 미팅 시간 정보를 저장해야 한다
- 사용자는 전사기록을 확인할 수 있어야 한다

### AI 요약 기능

- 시스템은 전사된 기록을 5줄로 간략 요약해야 한다
- 시스템은 상위 5개 키워드를 추출해야 한다
- 사용자는 AI 요약 결과를 확인할 수 있어야 한다

### 언어 Confidence 필터

- 시스템은 언어 Confidence값을 확인하여 전사 후 필터링해야 한다
- 시스템은 같은 공간에서 다른 언어로 말할 시 섞여서 들어가는 것을 방지해야 한다

### 마이크 제어

- 호스트는 마이크를 ON/OFF할 수 있어야 한다
- 게스트는 마이크를 ON/OFF할 수 있어야 한다
- 시스템은 하드웨어적 마이크 제어를 지원해야 한다
- 시스템은 인앱 마이크 제어 기능을 제공해야 한다

### 인원 관리

- 호스트는 1:1 서비스 미팅방을 인원 제한 없이 운영할 수 있어야 한다
- 시스템은 N:N 대화를 지원해야 한다
- 호스트는 게스트를 퇴장시킬 수 있어야 한다

### 미팅 진행

- 호스트는 태블릿으로 상담을 진행할 수 있어야 한다
- 호스트는 모바일앱으로 이동하며 설명할 수 있어야 한다
- 게스트는 모바일앱으로 참여할 수 있어야 한다
- 게스트는 스마트안경으로 참여할 수 있어야 한다

## 5. 양자회담 (N:N) 관련 스토리

### 다중 참가자 관리

- 여러 호스트가 각자 태블릿을 사용할 수 있어야 한다
- 여러 게스트가 각자 태블릿을 사용할 수 있어야 한다
- 시스템은 다대다 회의를 지원해야 한다
- 시스템은 상담 기능 내 인원 제한을 해제할 수 있어야 한다

## 6. 관리자 페이지 관련 스토리

### 상담록 생성

- 시스템은 AI를 활용하여 상담록을 생성해야 한다
- 시스템은 표준 상담 양식에 맞게 정보를 추출해야 한다
- 관리자는 상담록을 확인할 수 있어야 한다
- 관리자는 상담록을 다운로드할 수 있어야 한다

### 세션 통합 관리

- 관리자는 진행실에서 PC앱으로 여러 세션을 통합 관리할 수 있어야 한다
- 관리자는 하나의 방에서 여러 세션을 생성할 수 있어야 한다
- 관리자는 PPT 전체화면에 번역 자막을 제공할 수 있어야 한다

### 데이터 분석

- 관리자는 데이터 기반 이벤트 성과를 분석할 수 있어야 한다
- 관리자는 개선 인사이트를 확인할 수 있어야 한다
- 관리자는 참가자 만족도를 분석할 수 있어야 한다

## 7. 기술 및 정책 관련 스토리

### 데이터 보관 정책

- 시스템은 대화 기록을 1년간 보관해야 한다
- 시스템은 업체 요청 시 대화 기록을 삭제해야 한다
- 사용자는 본인의 대화 기록을 삭제 요청할 수 있어야 한다

### 권한 관리

- 시스템은 마이크 권한을 관리해야 한다
- 시스템은 호스트 우선순위를 적용해야 한다
- 시스템은 게스트 제한을 적용해야 한다

### 디바이스 호환성

- 시스템은 무선 마이크 연결 시 OS 설정을 따라야 한다
- 시스템은 입력 장치 우선순위를 적용해야 한다
- 시스템은 물리적 마이크 버튼과 앱 UI를 연동해야 한다

### 성능 관리

- 시스템은 동시접속자 수 임계값을 관리해야 한다
- 시스템은 서버 부하를 모니터링해야 한다
- 시스템은 실시간 번역 품질을 유지해야 한다

## 8. 사용자 경험 관련 스토리

### 접근성

- 모든 사용자는 시각 장애인을 위한 음성 안내를 받을 수 있어야 한다
- 모든 사용자는 청각 장애인을 위한 시각적 피드백을 받을 수 있어야 한다

### 오류 처리

- 사용자는 네트워크 오류 시 적절한 안내를 받을 수 있어야 한다
- 사용자는 마이크 권한 오류 시 해결 방법을 안내받을 수 있어야 한다
- 사용자는 번역 오류 시 재시도할 수 있어야 한다

### 사용성 개선

- 사용자는 자막 크기를 조절할 수 있어야 한다
- 사용자는 자막 색상을 변경할 수 있어야 한다
- 사용자는 번역 속도를 조절할 수 있어야 한다
